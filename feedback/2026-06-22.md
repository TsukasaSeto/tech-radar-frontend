### 改善が必要な点・観察事項

- **Playwright sabotage method の位置づけ**: Rule #1（E2Eテストの設計原則）への強化として追加したが、独立したルール（Rule #X: 破壊テストで劇場テストを検出する）の方が発見しやすかったかもしれない。次回同様のトピックが来たら切り出しを検討する。
- **hook latency Rule の確信度**: 単一 Zenn 記事（bokuwalily）のみのため確信度「中」。公式ドキュメントや複数記事での裏付けがあれば「高」に格上げできる。6ヶ月後の見直し候補。
- **CSS Anchor Positioning のスキップ**: 現時点では実験的 API だが、ブラウザサポートが進んだ時点で採用を検討する。`_seen.json` に記録済みのため、再登場時に changelog で確認できる。

### プロンプト改善提案

- **TypeScript RC フェーズの採用条件を明示**: 現行プロンプトは「安定版のみ採用」という暗黙のルールがあるが、RC + 複数独立記事の組み合わせで採用を許可する条件をプロンプトに明示すると、次回から迷わず判断できる。
  - 提案文言: `RC版は Pattern 2（2記事以上）かつ 移行リスクを Stage 1〜4 段階で示せる場合に限り採用可。確信度は最大「中」とする。`
