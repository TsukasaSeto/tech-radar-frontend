## プロンプト改善提案（1件）

### 1. Pattern 2 の代替条件として「公式ドキュメントへの言及 + 実践コード例」を認める
- **対象セクション**: 「採用基準」セクション — Pattern 2 の条件
- **現状の記述**: "コミュニティ記事 2件以上 + 実践コード例"
- **提案する変更**: 以下の代替条件を追加:
  ```
  Pattern 2 代替条件:
  - 公式ドキュメント（MDN / W3C / RFC / OSS 公式）への直接引用 + 実践コード例
    （単一コミュニティ記事でも公式仕様の実装解説であれば採用可）
  ```
- **理由**: t3-env+Valibot（Vite向け型安全 env）や CSS scroll-padding-inline のように、公式仕様を実践的に解説した単一記事が Pattern 2 の「2件以上」条件に引っかかって採用できないケースが多い。公式ドキュメント引用がある場合は信頼性が担保されるため、1件でも採用可とするのが合理的。
- **想定リスク**: 採用件数が増えすぎて PR がブロートする可能性。採用 1 回あたりの文量制限（例: 本文 4 行以内）と組み合わせる。
- **優先度**: 中
