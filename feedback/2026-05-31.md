## プロンプト改善提案（1件）

### 1. Routine 開始 UTC 時刻を環境変数または実行ログから取得する手順を追加
- **対象セクション**: 「期間とスコープ」セクション
- **現状の記述**: "基準時刻: Routine 実行開始時の UTC 時刻"（取得方法の記述なし）
- **提案する変更**: 以下を追記する:
  ```
  **実行開始時刻の取得**:
  ```bash
  ROUTINE_START_UTC=$(date -u +%Y-%m-%dT%H:%M:%SZ)
  CUTOFF_UTC=$(date -u -d "$ROUTINE_START_UTC - 36 hours" +%Y-%m-%dT%H:%M:%SZ)
  echo "36h window: $CUTOFF_UTC 以降"
  ```
  この変数を Step 6 のフィルタ判定に使用する。
  ```
- **理由**: 現状は「Routine 実行開始時の UTC 時刻」が暗黙的で、フィルタ判定がモデルの推定に依存している。コマンドで明示的に計算することで再現性が高まる。
- **想定リスク**: cloud env で `date` コマンドが使えない場合は bash ではなく Python の `datetime.utcnow()` に差し替える必要がある。
- **優先度**: 中
