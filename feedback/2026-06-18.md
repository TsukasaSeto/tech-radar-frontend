## プロンプト改善提案（2件）

### 1. `git diff --cached` チェックを URL 単位の検証に変更する
- **対象セクション**: `_seen.json の仕様 > 削除禁止ルール > 5. commit 直前にもう一度確認`
- **現状の記述**: `git diff --cached _seen.json | grep '^-  "' && { echo "ERROR: URL deletion in staged diff"; exit 1; }`
- **提案する変更**: `diff <(git show HEAD:_seen.json | jq -r '.[]' | sort) <(jq -r '.[]' _seen.json | sort) | grep '^<' && { echo "ERROR: URL deletion detected"; exit 1; }` に変更する
- **理由**: 現行パターンは JSON フォーマット変更（末尾要素へのカンマ追加）を URL 削除として誤検知する。jq を使った sorted 比較なら実際の URL 削除のみを検知できる
- **想定リスク**: git show が若干遅くなる程度、実質的なリスクなし
- **優先度**: 中

### 2. Stripe Engineering を「時間制約時はスキップ可」と明記する
- **対象セクション**: `Step 5: HTML特殊処理ソースの取得（Stripe Engineering）`
- **現状の記述**: 「HTML特殊処理ソースの取得（Stripe Engineering）」として必須ステップ扱い
- **提案する変更**: "月0〜1記事ペースのため、全ソース処理後に時間が余った場合のみ試行する。時間制約時はスキップして changelog に「Stripe: 時間制約スキップ」と記録する" を追加
- **理由**: Stripe Engineering は月0〜1記事で取得成功しても新着0件が多く、コストパフォーマンスが低い。時間を他のより活発なソースに使う方が合理的
- **想定リスク**: Stripe が稀に出す高品質記事を見逃すリスクがあるが、週次ルーティンであれば次週で拾える
- **優先度**: 低

- **種類**: ツール挙動
