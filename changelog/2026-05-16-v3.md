## 採用・強化の詳細

### A: 新規採用

#### Rule #9（新規） — `observability/error-tracking.md`

**ルール名**: Vercel Protected Source Maps でブラウザソースマップを認証保護する

- **採用パターン**: Pattern 1（公式ブログ/Changelog — 単一ソースで採用）
- **出典**: [Protected Source Maps: Ship browser source maps securely](https://vercel.com/changelog/protected-source-maps-ship-browser-source-maps-securely) (Vercel Changelog, 2026-05-16)
- **判断理由**: Vercel 公式 Changelog に掲載された新機能。既存 Rule #4（Source Map を Sentry にアップロード）と補完関係にあり、デプロイ済みソースマップの外部公開リスクという別の観点をカバーする独立したルールとして採用。

---

### B: 既存ルール強化

#### Rule #4 強化 — `performance/bundle-size.md`

**対象ルール**: 「`import()` による動的インポートと Route-based Code Splitting を実装する」

- **出典**: [[Frontend Performance - Part 16] 配信最適化の総仕上げ：Code Splitting・Cache・CDN戦略まとめ](https://qiita.com/tuanphan/items/e332db235525cfb9d10b) (Qiita tuanphan / 2026-05-15)
- **追加内容**:
  - コード分割の過剰適用アンチパターン（5KB×100チャンク問題、モバイルネットワークでの遅延）
  - 分割粒度の優先順位（ルート > 機能モジュール > コンポーネント）
  - HTTP キャッシュ推奨設定（静的アセット: `max-age=31536000,immutable` / HTML: `no-cache` / API: `stale-while-revalidate=60`）
