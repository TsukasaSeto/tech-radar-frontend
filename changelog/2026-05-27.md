## 追加・強化されたルール

### 新規 (2件)
1. `observability/tracing.md` - Rule #5: Sentry と OpenTelemetry を排他的でなく補完的に統合する
   - 出典: [How Sentry and OpenTelemetry Work Together](https://blog.sentry.io/sentry-opentelemetry-work-together/) ※2026-05-27に実際にfetch成功（Sentry公式ブログ）
   - 引用: > "keep OpenTelemetry where it already works, add the Sentry SDK where it gives you more application context"
   - 要点: `propagateTraceparent: true` + `tracePropagationTargets` 設定 + バックエンド CORS ヘッダー許可の3点セット

2. `ai-agent/claude-code.md` - Rule #17: AI エージェントに本番エラーのコンテキストを渡し、自己修正サイクルを構成する
   - 出典: [Agents Need Production Context](https://blog.sentry.io/agents-need-production-context/) ※2026-05-27に実際にfetch成功（Sentry公式ブログ）
   - 引用: > "The bottleneck for self-healing software isn't agent intelligence. It's that agents have no idea what actually broke."
   - 要点: Sentry MCP (`npx add-mcp https://mcp.sentry.dev/mcp`) でエラー詳細をエージェントに取り込み、draft PR として出力

### 強化 (1件)
3. `security/auth-token-storage.md` - Rule #1: NextAuth.js session() コールバックのアンチパターンを追加
   - 出典: [セッションとJWTとCookieの歴史と変遷から紐解く、認証トークン管理設計](https://zenn.dev/khale/articles/web-session-jwt-cookie-history) ※2026-05-27に実際にfetch成功
   - 引用: > "XSS をゼロにできない以上、『漏れたら終わり』のトークンを可読領域に置くこと自体が設計上の妥協です"
   - 統合先: session() コールバックアンチパターンコード例 + BFF 分離パターンコード例 + 出典追加
